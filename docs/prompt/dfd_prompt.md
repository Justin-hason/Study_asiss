# Study_asiss 项目 DFD（数据流图）绘制提示词

## 项目背景
Study_asiss 是一个智能学习助手系统，提供文档管理、知识提取、智能问答、练习题生成等功能。系统采用前后端分离架构，后端使用FastAPI，前端使用React。

## 系统功能模块

### 1. 用户认证与授权模块
**功能描述**：处理用户注册、登录、权限验证
**主要功能**：
- 用户注册
- 用户登录（JWT认证）
- 获取当前用户信息
- 角色权限控制（user/admin/auditor）

**数据流**：
- 输入：用户名、密码、邮箱
- 处理：密码哈希、JWT令牌生成、权限验证
- 输出：JWT令牌、用户信息

### 2. 文档管理模块
**功能描述**：处理文档的上传、解析、版本管理、分享
**主要功能**：
- 文档分块上传（初始化、上传分块、完成上传）
- 文档解析（DOCX、PDF、PPTX、TXT）
- 文档版本管理
- 文档预览
- 文档删除
- 文档权限管理
- 文档分享（创建分享链接、分享审核）

**数据流**：
- 输入：文件、分块数据、权限设置、分享请求
- 处理：文件存储、内容解析、Markdown转换、权限验证
- 输出：文档信息、预览内容、分享链接

### 3. 知识报告生成模块
**功能描述**：基于文档生成知识体系报告
**主要功能**：
- 生成知识报告（支持多文档融合）
- 重新生成报告
- 报告下载（Markdown/JSON）
- 报告删除
- 报告分享审核
- 公共报告管理

**数据流**：
- 输入：文档ID列表、标题、描述
- 处理：文档内容提取、AI生成报告、格式化输出
- 输出：知识报告（Markdown/JSON）

### 4. 智能问答模块
**功能描述**：基于文档内容和联网搜索回答用户问题
**主要功能**：
- 提交问题
- 检索相关文档
- 联网搜索补充
- AI生成答案
- 问答会话管理

**数据流**：
- 输入：用户问题
- 处理：文档检索、联网搜索、AI生成
- 输出：答案、来源引用

### 5. 练习题生成模块
**功能描述**：基于文档生成练习题并管理答题记录
**主要功能**：
- 开始练习（生成题目）
- 提交答案
- 完成练习
- 练习历史查询
- 错题本管理

**数据流**：
- 输入：文档ID、题目数量
- 处理：文档内容提取、AI生成题目、答案验证
- 输出：练习题、答题结果、统计信息

### 6. 知识库管理模块
**功能描述**：管理知识点和学习路径
**主要功能**：
- 知识点提取
- 知识树管理
- 知识点关联

**数据流**：
- 输入：文档内容
- 处理：AI提取知识点、构建知识树
- 输出：知识点列表、知识树结构

### 7. 学习统计模块
**功能描述**：统计学习数据和分析学习效果
**主要功能**：
- 学习时长统计
- 文档阅读统计
- 练习成绩统计
- 学习趋势分析

**数据流**：
- 输入：用户ID、时间范围
- 处理：数据聚合、统计分析
- 输出：统计报告、图表数据

## 外部实体

### 1. 用户（User）
- 描述：使用系统的终端用户
- 交互：
  - 提交登录/注册请求
  - 上传文档
  - 查看文档
  - 提交问题
  - 进行练习
  - 查看统计

### 2. 管理员（Admin）
- 描述：系统管理员
- 交互：
  - 审核分享请求
  - 管理公共资源
  - 查看系统统计

### 3. 审核员（Auditor）
- 描述：内容审核员
- 交互：
  - 审核文档分享请求
  - 审核知识报告分享请求

### 4. AI服务（AI Service）
- 描述：DeepSeek大模型服务
- 交互：
  - 接收生成请求
  - 返回生成结果

### 5. 搜索服务（Search Service）
- 描述：Tavily联网搜索服务
- 交互：
  - 接收搜索查询
  - 返回搜索结果

### 6. 文件存储（File Storage）
- 描述：本地文件系统
- 交互：
  - 存储上传文件
  - 读取文件内容

### 7. 数据库（Database）
- 描述：PostgreSQL数据库
- 交互：
  - 存储用户数据
  - 存储文档信息
  - 存储学习记录

## 零层DFD（Context Diagram）绘制要求

### 绘制元素
1. **中心处理**：Study_asiss系统
2. **外部实体**：
   - 用户
   - 管理员
   - 审核员
   - AI服务
   - 搜索服务
   - 文件存储
   - 数据库

### 数据流
1. 用户 → 系统：
   - 登录/注册信息
   - 上传文件
   - 查询请求
   - 问题
   - 答案

2. 系统 → 用户：
   - JWT令牌
   - 文档信息
   - 查询结果
   - 答案
   - 统计报告

3. 系统 → AI服务：
   - 生成请求（报告、题目、答案）

4. AI服务 → 系统：
   - 生成结果

5. 系统 → 搜索服务：
   - 搜索查询

6. 搜索服务 → 系统：
   - 搜索结果

7. 系统 ↔ 文件存储：
   - 文件读写

8. 系统 ↔ 数据库：
   - 数据读写

### 布局要求
- 系统位于中心
- 外部实体围绕系统分布
- 数据流使用箭头标注方向
- 使用不同颜色区分不同类型的数据流

## 一层DFD（Level-1 DFD）绘制要求

### 主要处理过程

#### 1. 认证处理（Authentication Process）
- 输入：用户名、密码
- 处理：验证用户、生成JWT
- 输出：JWT令牌
- 存储：用户信息（数据库）

#### 2. 文档处理（Document Process）
- 输入：上传文件、分块数据
- 处理：文件存储、内容解析、版本管理
- 输出：文档信息、预览内容
- 存储：文件（文件存储）、文档元数据（数据库）

#### 3. 知识报告处理（Knowledge Report Process）
- 输入：文档ID、生成参数
- 处理：内容提取、AI生成、格式化
- 输出：知识报告
- 存储：报告内容（数据库）
- 外部调用：AI服务

#### 4. 问答处理（QA Process）
- 输入：用户问题
- 处理：文档检索、联网搜索、AI生成
- 输出：答案、来源
- 存储：问答记录（数据库）
- 外部调用：AI服务、搜索服务

#### 5. 练习处理（Practice Process）
- 输入：文档ID、题目数量
- 处理：题目生成、答案验证、统计
- 输出：练习题、答题结果
- 存储：练习记录（数据库）
- 外部调用：AI服务

#### 6. 知识库处理（Knowledge Base Process）
- 输入：文档内容
- 处理：知识点提取、知识树构建
- 输出：知识点列表、知识树
- 存储：知识点（数据库）
- 外部调用：AI服务

#### 7. 统计处理（Statistics Process）
- 输入：用户ID、查询参数
- 处理：数据聚合、统计分析
- 输出：统计报告
- 读取：学习记录（数据库）

### 数据存储
1. **用户数据库（User DB）**：存储用户信息、权限
2. **文档数据库（Document DB）**：存储文档元数据、版本
3. **知识数据库（Knowledge DB）**：存储知识报告、知识点
4. **练习数据库（Practice DB）**：存储练习题、答题记录
5. **问答数据库（QA DB）**：存储问答会话、消息
6. **文件存储（File Storage）**：存储上传文件

### 数据流关系

#### 认证处理
- 输入：用户 → 认证处理（登录/注册信息）
- 输出：认证处理 → 用户（JWT令牌）
- 存储：认证处理 ↔ 用户数据库（用户信息）

#### 文档处理
- 输入：用户 → 文档处理（上传文件）
- 输出：文档处理 → 用户（文档信息）
- 存储：文档处理 ↔ 文档数据库（文档元数据）
- 存储：文档处理 ↔ 文件存储（文件内容）

#### 知识报告处理
- 输入：用户 → 知识报告处理（生成请求）
- 输出：知识报告处理 → 用户（知识报告）
- 存储：知识报告处理 ↔ 知识数据库（报告内容）
- 外部：知识报告处理 ↔ AI服务（生成请求/结果）

#### 问答处理
- 输入：用户 → 问答处理（问题）
- 输出：问答处理 → 用户（答案）
- 存储：问答处理 ↔ 问答数据库（问答记录）
- 外部：问答处理 ↔ AI服务（生成请求/结果）
- 外部：问答处理 ↔ 搜索服务（搜索查询/结果）

#### 练习处理
- 输入：用户 → 练习处理（开始练习）
- 输出：练习处理 → 用户（练习题、结果）
- 存储：练习处理 ↔ 练习数据库（练习记录）
- 外部：练习处理 ↔ AI服务（生成请求/结果）

#### 知识库处理
- 输入：用户 → 知识库处理（文档内容）
- 输出：知识库处理 → 用户（知识点）
- 存储：知识库处理 ↔ 知识数据库（知识点）
- 外部：知识库处理 ↔ AI服务（提取请求/结果）

#### 统计处理
- 输入：用户 → 统计处理（查询参数）
- 输出：统计处理 → 用户（统计报告）
- 读取：统计处理 → 用户数据库（用户信息）
- 读取：统计处理 → 文档数据库（文档统计）
- 读取：统计处理 → 练习数据库（练习统计）

## 绘制规范

### 符号说明
- **圆角矩形**：处理过程（Process）
- **矩形**：外部实体（External Entity）
- **开口矩形**：数据存储（Data Store）
- **箭头**：数据流（Data Flow）

### 命名规范
- 处理过程：动词+名词（如"认证处理"、"文档处理"）
- 数据流：名词短语（如"登录信息"、"文档内容"）
- 数据存储：名词+数据库（如"用户数据库"、"文档数据库"）

### 颜色规范
- 处理过程：蓝色
- 外部实体：绿色
- 数据存储：橙色
- 数据流：灰色

### 布局规范
- 零层DFD：系统居中，外部实体围绕分布
- 一层DFD：按功能模块分组，数据存储位于底部
- 数据流避免交叉，保持清晰

## 输出格式
请使用 Mermaid DFD 语法或专业的DFD工具（如draw.io、Lucidchart）绘制零层和一层DFD。

## 注意事项
1. 零层DFD展示系统与外部实体的交互
2. 一层DFD展示系统内部的主要处理过程
3. 数据流需要标注清晰的数据内容
4. 处理过程需要编号（P1、P2、P3...）
5. 数据存储需要编号（D1、D2、D3...）
6. 外部实体需要编号（E1、E2、E3...）
7. 保持数据流的平衡性
8. 避免数据流交叉和混乱
